Cozy brings to you its personal cloud platform. Developing your node.js personal web apps has never been that easy ; get ready in 10minutes and start leveraging your personal data your way!

Photos, mails, notes, contacts, todos... store and manage any kind of data. There is an app for an usage. And if you don't find the app you want, you can build it!

Cozy is open-source and aims to be self-hosted. That means it is privacy-safe. If you don't want to be bored with installation, check our hosting solutions.

host it

  Host it

curl http://cozy.io/install_cozy.sh | HOST=root@ip:port
read more

This script will install several tools on your local computer: python, pip, fabric and fabtools. They are the tools required to run the installation on your remote server. To know what will be setup on your server, check the list in our readme.

It will work only with Debian/Ubuntu distros. A fully packaged VM is on its way so that you can deploy a Cozy on any OS (Linux, Mac, Win).

hack it

  Hack it

install the development environment

npm install cozy -g cozy dev:init cozy dev:start

initialize your application

cozy new my-app --github myaccount cd my-app

hack then deploy !

cozy deploy

delete it

  Delete it

We are convinced that you will keep using your Cozy because you can get rid of it:

curl http://cozy.io/uninstall_cozy.sh | HOST=root@ip:port sh
read more

Be aware that with this action, you will lost all your data permanently.

see our architecture

  Architecture

proxy

The proxy handles all users requests. It manages the authentication and route requests to the right application.

home / controller

The home is an app that acts as the UI for the controller. It allows to install or remove applications and sets user parameters.

app1

This is an application provided by Cozy Cloud or the community. It is spawned by the which find the sources on a github repo

app2

Another app: notes, mails, budget, sport coach...

...
your app

This is your application. You can spawn it like any other applications.

data system

The data system is a layer between apps and data. It (will) manages application permissions and provides an unified API to access to all kind of data: files, documents or indexes. It publishes changes, this way, applications can communicate with each other.

database

This NoSQL storage stores documents as JSON objects.

file system

Files are stored directly in the database as attachments of documents. A fuse implementation is on the roadmap to make things smoother.

indexer

A dedicated module handles content indexation through a REST API. That's your personal search engine.
