  Host it

Target your server with this script (sudoer required).

curl http://cozy.io/install_cozy.sh | HOST=root@ip sh

 

This script installs several tools on your local computer: python, pip, fabric and fabtools. They are required to run the installation on your remote server. The script works only with Debian/Ubuntu distros.

Check our  full documentation

Painless web development

  • Use our ready to-go templates and easy deployment for your web app
  • Write web apps that leverage user personal data
  • Distribute them quickly to the community
 

  Hack it

install the development environment (node.js, vagrant and virtualbox are required)

npm install cozy -g
cozy dev:init
cozy dev:start

initialize your application

cozy new my-app --github myaccount
cd my-app

hack... then deploy!

cozy deploy

When you're done send us an email and we'll help you to promote your app!

For more details check our tutorial

You will stay because you can leave.

  • You can delete your Cozy and all your data when you want

  Delete it

curl http://cozy.io/uninstall_cozy.sh | HOST=root@ip:port sh

What is annoying nowadays with online services is that you are locked with them. We don't want that for Cozy Cloud. That's why we give you the possibility to leave the platform.
NB: Be aware that the above action will erase your data permanently.

Cozy is a personal PaaS

  • It manages web apps and offer a JSON data store, an indexer and a file system
  • All of them are abstracted by a module called the Data System

  Architecture

proxy

The proxy handles all user requests. It manages the authentication and routes requests to the right application.

home / controller

The home app is an UI for the controller. It allows to install or remove applications and to set user parameters.

app1

This is an application provided by Cozy Cloud or the community. It is spawned by the home that finds the sources on a github repo.

app2

Your choice: notes, mails, budget, sport coach...

...
your app

This is an application you developed. You can spawn it like any other applications.

data system

The data system is a layer between apps and data. It (will) manages application permissions and provides a unified API to access to all kind of data: files, documents or indexes. It publishes changes, so that applications can share data with each other.

database

This NoSQL storage stores documents as JSON objects.

file system

Files are stored directly in the database as attachments of documents. A fuse implementation is on the roadmap to make things smoother.

indexer

A dedicated module handles content indexation through a REST API. It's your personal search engine.

If you want to know more about our architecture check the full documentation
